renderForm SDK Reference

A simplified entry point for rendering AEM Adaptive Forms directly from a form definition object — no DOM block boilerplate required.

async renderForm(formDef, element)

import { renderForm } from '/blocks/form/form.js';

Renders a form definition into a target DOM element. Handles both AEM (adaptive) and document-based (spreadsheet) form types automatically. Initialises the rule engine, registers custom functions, and wires all event handling.

ParameterTypeDescription
formDefObjectParsed form definition JSON (adaptive form schema).
elementHTMLElementTarget container element. The rendered <form> is appended here.

Returns Promise<{ form, afbForm }>

PropertyTypeDescription
formHTMLFormElementThe rendered form element. Has dataset.id, dataset.action, dataset.source, dataset.rules.
afbFormFormModel | nullThe live form model instance. null when using Web Worker mode (model lives in worker thread).
import { renderForm } from '/blocks/form/form.js';

const container = document.getElementById('my-container');
const { form, afbForm } = await renderForm(formDef, container);

// form is now in the DOM
console.log(form.dataset.id);   // form ID
console.log(form.dataset.source); // 'aem' or 'sheet'

Form Model API (afbForm)

The afbForm instance is the live rule-engine model. Available when the form runs without a Web Worker (e.g. environments where typeof Worker === 'undefined').

afbForm.exportData() → Object

Returns the current form data as a plain object, keyed by field name. Excludes disabled and unbound (dataRef: null) fields.

const data = afbForm.exportData();
// { firstName: 'Jane', email: 'jane@example.com', ... }
afbForm.importData(data: Object) → void

Prefills the form with the provided data object. Keys must match field name values. Triggers change events and rule re-evaluation for each updated field.

afbForm.importData({
  firstName: 'Jane',
  email: 'jane@example.com',
  country: 'US',
});
afbForm.validate() → ValidationError[]

Runs validation on all fields. Returns an array of validation error objects — empty array means the form is valid.

const errors = afbForm.validate();
if (errors.length === 0) {
  console.log('Form is valid');
} else {
  errors.forEach(e => console.log(e.fieldName, e.message));
}
afbForm.getState(includeData?: boolean) → Object

Returns a serialisable snapshot of the current form state. Pass true to include current field values in the snapshot.

const state = afbForm.getState(true);
console.log(state.items); // all field states
afbForm.getElement(fieldId: string) → FieldModel

Returns the model for a single field by its id or qualifiedName. Useful for reading or imperatively setting a field's properties.

const emailField = afbForm.getElement('emailinput-abc123');
console.log(emailField.value);      // current value
console.log(emailField.valid);      // validation state
emailField.value = 'new@email.com'; // set value directly
afbForm.subscribe(callback, eventType?: string) → void

Subscribes to model events. If eventType is omitted, receives all events.

Event typeWhen it fires
'fieldChanged'Any field property changed (value, visible, enabled, enum…)
'change'A field's value changed
'submitSuccess'Form submitted successfully
'submitFailure'Submission rejected by server
'submitError'Network or unexpected error on submit
afbForm.subscribe((event) => {
  const { field } = event.payload;
  console.log(`${field.name} changed:`, field.value);
}, 'fieldChanged');

// Listen to value changes only
afbForm.subscribe((event) => {
  console.log('value changed', event.payload);
}, 'change');
afbForm.dispatch(event: Event) → void

Dispatches a custom event into the rule engine. Used to trigger custom event handlers defined in the form rules.

// Trigger a custom event on the form
afbForm.dispatch(new CustomEvent('custom:refresh'));

// Trigger click on a specific field
const btn = afbForm.getElement('submit-button-id');
btn.dispatch({ type: 'click' });
afbForm.reset() → void

Resets all fields to their initial values as defined in the form definition. Equivalent to the HTML form reset but operates through the model so rules re-evaluate.

afbForm.reset();

Component Subscribe Helper

import { subscribe } from '/blocks/form/rules/index.js';

A higher-level helper for custom components that need to react to field model changes after the DOM has been rendered. Called inside a component decorator.

subscribe(fieldDiv, formId, callback, options?) → void
ParameterDescription
fieldDivThe field's wrapper DOM element (data-id must be set).
formIdThe form's dataset.id value.
callback(div, model, eventType, payload)Called with 'register' once on init, then 'change' on each value update.
options.listenChangesSet true to receive 'change' callbacks after registration.
// Inside a component decorator:
export default async function decorateRating(element, fd, container, formId) {
  subscribe(element, formId, (div, model, eventType, payload) => {
    if (eventType === 'register') {
      renderStars(div, model.value);
    }
    if (eventType === 'change') {
      renderStars(div, payload.changes[0].currentValue);
    }
  }, { listenChanges: true });
}
